# CPR 语音 Copilot 实验

这是一个面向成人 CPR 语音引导研究的网页原型。
它不是商业急救产品，也不是纯问卷工具，而是为了研究：

- 不同解释深度的语音引导，如何影响参与者的行动启动
- 不同安抚方式，如何影响参与者在害怕、不确定、被打断时的恢复执行
- 不同时间压力条件下，参与者对系统的信任是否更恰当，而不是单纯更高

## 这版研究的核心问题

这版 prototype 主要回答三个问题：

1. 只给指令、给简短解释、给完整解释时，参与者的启动速度和执行过程会不会不同。
2. 当参与者说“我很害怕”或“我不确定”时，系统的安抚是否能帮助他们继续完成 CPR 流程。
3. 在高压情境下，语音引导能否让参与者形成更合适的信任，而不是盲目依赖或过度怀疑。

所以，这个原型研究的重点是：

- 语音引导如何影响人的行动
- 安抚语句如何帮助恢复执行
- 解释深度如何影响信任校准

不是：

- 摄像头动作识别准不准
- 做一个真实医疗产品
- 做成纯问卷系统

## 当前原型包含什么

- 中文默认界面
- 英文可切换界面
- 成人 CPR 主场景
- 三层解释深度
- 多种时间压力条件
- “我很害怕 / 我不确定”安抚触发
- 实时日志
- 回放与标注
- JSON / CSV 导出
- 草稿保存与恢复
- 手机端自适应
- Safari 兼容

## 研究变量

### 1. 解释深度

- 对照组
- 纯指令型
- 简释型
- 全释型

### 2. 时间压力

- 低压
- 高压
- 隐藏倒计时

### 3. 安抚触发

- “我很害怕”
- “我不确定”
- 请求更多解释

### 4. 结果记录

- 首次响应时间
- 每步完成时间
- 总流程完整度
- 错误步骤
- 中断次数
- 纠错接受率
- 安抚请求次数
- 解释请求次数
- 信任与认知负荷问卷

## 适合怎么做实验

建议先只做一个主场景：

- 成人 CPR / 疑似心脏骤停

建议先做最稳的对比：

- 纯指令型
- 简释型
- 全释型
- 对照组

建议先测的主结果：

- 参与者是否更快开始动作
- 被打断后能否恢复执行
- 是否更愿意接受系统纠错
- 是否形成更恰当的信任

## 如何运行

在项目目录中启动本地服务：

```powershell
node server.mjs
```

然后打开：

- [http://127.0.0.1:3000](http://127.0.0.1:3000)

如果要在 iPhone 上打开：

- 手机和电脑连接同一个 Wi-Fi
- 在 iPhone Safari 中打开终端打印出来的局域网地址
- 如果打不开，优先检查电脑防火墙和路由器隔离

## 说明

这个系统是为研究服务的，不是医疗替代方案。
真实急救场景中，请优先拨打当地急救电话并遵循专业急救规范。

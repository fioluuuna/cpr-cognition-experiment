# CPR Cognition Experiment 实验员操作手册

这是给实验员和队友用的快速手册。  
当前版本是一个**网页版 pilot**，适合先在电脑和 iPhone Safari 上做完整试跑。

---

## 1. 启动方式

在项目目录执行：

```powershell
node server.mjs
```

启动后，终端会打印三个地址：

- `http://localhost:3000`
- 局域网地址，供同一 Wi-Fi 下的手机访问
- 如果有多个网卡，可能会显示多个地址，优先选当前连着 Wi-Fi 的那个

---

## 2. 参与者测试顺序

建议按下面顺序跑一轮完整测试：

1. 打开 **Setup**
2. 填参与者编号和 trial 编号
3. 选择解释深度、时间压力、场景版本
4. 点 **Start Experiment**
5. 到 **Participant Flow** 页做：
   - 同意
   - 设备检查
   - 训练示例
   - 进入正式任务
6. 到 **Task** 页完成 5 步 CPR 场景
7. 观察中断、错误、纠错、恢复是否正常记录
8. 进入 **Questionnaire**
9. 填完后点 **Finish session**
10. 到 **Export** 页导出 JSON / CSV
11. 到 **Replay & Annotation** 页核对事件回放

---

## 3. 实验员控制台怎么用

控制台在 **Experimenter Console** 页。

### 你可以做的事

- 启动任务
- 暂停 / 恢复任务
- 结束任务
- 手动插入错误
- 手动插入中断
- 手动插入提醒
- 添加观察标注

### 建议使用方式

- 正式实验前，先在电脑端自己跑一次
- 手机端先看完整流程是否能走通
- 再换成 iPhone Safari 复测一次

---

## 4. 这版 pilot 记录什么

### 行为日志

- 页面切换
- 按钮点击
- 解释请求 / 重播 / 跳过
- 每一步的选择
- 超时
- 中断
- 恢复
- 纠错接受 / 拒绝
- 标注内容

### 结果数据

- 反应时间
- 完成时间
- 正确率
- 流程完整度
- 中断次数
- 纠错接受率
- 问卷评分
- 开放题备注

---

## 5. 导出文件怎么用

### JSON
适合保留完整会话数据，后面做分析最有用。

### CSV
适合快速看事件时间线，或者直接丢进表格软件。

### Replay JSON
适合后期做回放标注和人工编码。

---

## 6. iPhone 测试建议

1. 用 Safari 打开局域网地址
2. 不要一开始就切后台
3. 先跑完整流程，再试导出
4. 如果下载按钮不稳定，优先用 **Export JSON** 后看页面预览，再让服务器保存一份

---

## 7. 试跑时重点看什么

### 必查项

- 手机端布局是否挤压
- 按钮是否够大
- 倒计时是否可见或符合当前条件
- 任务步骤是否能顺利推进
- 暂停 / 恢复是否会把时间逻辑弄乱
- 回放是否能看到完整事件
- 导出文件是否能正常拿到

### 常见问题

- 如果 iPhone 下载失败，不一定是数据没生成，先看页面上的 JSON 预览和服务器日志
- 如果语音播报触发不了，先确认浏览器允许音频播放
- 如果服务 worker 不生效，不影响主流程，网页依然可用

---

## 8. 实验结束后你应该得到什么

每个 session 最少应能导出：

- 参与者信息
- 条件信息
- 事件日志
- 行为响应
- 问卷结果
- 研究员标注

这就足够后续做 pilot 分析了。
